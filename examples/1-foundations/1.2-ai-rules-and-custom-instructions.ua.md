# 1.2 AI Rules / Custom Instructions

## Яку проблему це вирішує

У [1.1](./1.1-prompt-and-context-engineering.ua.md) я навчився передавати контекст у кожному промпті. Але *той самий* контекст — «це canvas, а не DOM-застосунок», «відступ 4 пробіли», «конфіг лежить у `config.ts`» — був потрібен майже в кожному промпті. Повторювати його марнотратно й легко забути, а забуте обмеження означає неправильну відповідь.

**Файли AI-правил** виносять ці стабільні обмеження з промпта в репозиторій, де агент завантажує їх автоматично на кожен запит.

## Що я додав у `memory-cards`

Один файл [`AGENTS.md`](https://github.com/smnvsergii/phaser-memory-cards-MFE/blob/main/AGENTS.md) у корені репозиторію.

### Чому `AGENTS.md`, а не `.cursorrules`

Форматів кілька: `.cursorrules` (Cursor), `.github/copilot-instructions.md` (Copilot), `CLAUDE.md` (Claude Code) і tool-agnostic `AGENTS.md`. Я обрав **один канонічний `AGENTS.md`** із двох причин:

1. **Уникнути рассинхрону.** Кілька файлів правил неминуче розходяться. Одне джерело істини — у цьому й сенс.
2. **Портативність.** `AGENTS.md` — відкрита конвенція, яка набирає обертів і яку читає більшість агентів; якщо конкретному інструменту потрібне власне ім'я файлу, я зробив би його тонким покажчиком на `AGENTS.md`, а не дублював вміст.

> 💡 Висновок: стався до AI-правил як до будь-якого спільного конфігу — одне джерело істини, без копій, що можуть розійтися.

## Як AI-інструменти насправді його знаходять

Справедливе питання: *якщо файл один, як його бачить кожен інструмент?* У 2024–2025 кожен інструмент мав власне ім'я файлу (`.cursorrules`, `.github/copilot-instructions.md`, `CLAUDE.md`, …). До 2026 індустрія зійшлася на **`AGENTS.md` у корені репозиторію** як відкритому стандарті (під опікою Linux Foundation / Agentic AI Foundation). Більшість агентів — Cursor, GitHub Copilot, Codex, Gemini CLI, Windsurf, Aider, Zed і 20+ інших — тепер читають його **нативно** на старті сесії.

Кілька інструментів усе ще очікують власне ім'я файлу. Для них я додаю **тонкий шим** — файл з одного рядка, що вказує назад на `AGENTS.md`, а не копію:

| Інструмент | Файл у репо | Роль |
| ---------- | ----------- | ---- |
| Більшість агентів | `AGENTS.md` | Джерело істини |
| Claude Code | `CLAUDE.md` | Шим → імпорт `@AGENTS.md` |
| Kiro | `.kiro/steering/project-rules.md` | Шим → включає `AGENTS.md` |

Приклад шима (`CLAUDE.md`):

```markdown
See @AGENTS.md for all project instructions.
```

Це зберігає властивість «одне джерело істини» й водночас покриває інструменти, які не читають `AGENTS.md` нативно. Механіку також задокументовано всередині самого `AGENTS.md` у секції *"How tools discover this file"*, тож кожен, хто відкриє репо, зрозуміє схему.

> 💡 Висновок: шими вказують, а не копіюють. Один файл містить правила; кожне інше ім'я файлу — це редирект на нього.

## Як структурований файл

Кожна секція існує, щоб вбити клас помилок, який модель реально робила.

| Секція | Яку помилку запобігає |
|--------|------------------------|
| **Project in one line** | Агент сприймає Phaser-canvas-гру як React/DOM-застосунок. |
| **Tech stack (не вводити альтернативи)** | Тягне React, CSS-фреймворк чи state-бібліотеку «щоб допомогти». |
| **Commands** | Вигадує назви скриптів замість реальних `npm run`. |
| **Before you finish (верифікація)** | Оголошує «готово» без запуску typecheck / lint / build. |
| **Code style** | Неправильні відступи/лапки, які pre-commit хук усе одно відхилить. |
| **Architecture & conventions** | Завантажує ассети не в тій сцені, хардкодить магічні числа, використовує CSS media queries для canvas-розкладки. |
| **MFE protocol** | Ламає discriminated union `MFEEnvelope` або мовчки розсинхронізовує з shell. |
| **Guardrails** | Редагує згенеровані теки, комітить без запиту, рефакторить сторонній код. |

### Принципи хороших правил

- **Роби правила перевірюваними, а не «на відчуття».** «Запусти `npm run typecheck`, `lint`, `build` перед завершенням» краще за «пиши хороший код».
- **Прив'язуй правила до реальних артефактів.** Посилайся на `config.ts`, `protocol.ts`, реальні налаштування Prettier — щоб агент міг себе перевірити.
- **Формулюй інваріанти явно.** Приклад: *переворот анімує лише `scaleX`, ніколи `scaleY`*. Це неочевидне правило, яке новий агент порушив би.
- **Тримай файл придатним для швидкого читання.** Короткі секції, таблиці, наказовий тон. Правила, які ніхто не читає (людина чи модель), — мертвий вантаж.
- **Кодуй «не роби».** Секція guardrails («не додавай залежності для того, що Phaser уже вміє») запобігає найдорожчим помилкам.

## До / після

**До (по промпту, з 1.1):** кожен запит мусив повторювати стек, стиль і «це canvas».

**Після:** промпт стискається до власне задачі —

```
Додай подію "hintUsed" у MFE-протокол з payload { remaining: number }.
```

— і агент уже знає, що треба слідувати патерну `MFEEnvelope`, тримати `protocol.ts` в синхроні, поважати strict TS і запускати верифікацію, бо `AGENTS.md` сказав це один раз.
