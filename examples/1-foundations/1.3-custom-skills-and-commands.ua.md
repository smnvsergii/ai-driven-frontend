# 1.3 Custom Skills / Commands

**🌐 Мова:** [English](./1.3-custom-skills-and-commands.md) · **Українська**

**Блок:** Foundations
**Практичний проєкт:** [`phaser-memory-cards-MFE`](https://github.com/smnvsergii/phaser-memory-cards-MFE) — гра «знайди пару» на Phaser 3 + Vite + TypeScript, яка також працює як мікрофронтенд.

> Мета цієї нотатки: правила (1.2) описують, *як* поводитися на кожній задачі. **Кастомні команди** пакують конкретну *повторювану задачу* в один виклик, тож багатокрокова й схильна до помилок процедура стає `/command`.

---

## Правила vs команди — у чому різниця

| | AI-правила (1.2) | Кастомні команди (1.3) |
|--|------------------|-------------------------|
| Область | Діють на **кожен** запит | Викликаються для **однієї конкретної** задачі |
| Форма | Пасивний контекст (`AGENTS.md`) | Активна іменована дія (`/add-mfe-message`) |
| Аналогія | Стандарт кодування проєкту | npm-скрипт / ціль Makefile проєкту |

Правила не дають агенту робити неправильне загалом. Команди кодують точні кроки повторюваної роботи, щоб не пояснювати їх щоразу.

## Повторювана задача, яку я автоматизував

У `memory-cards` додавання повідомлення в MFE-протокол — невелика, але **багатофайлова й легка для поломки** процедура. Додавання однієї команди торкається:

1. `src/mfe/protocol.ts` — розширити discriminated union `ShellCommand` / `MFEEvent`.
2. `src/mfe/bridge.ts` — додати тип у **рантайм-`Set`** `SHELL_COMMAND_TYPES`.
3. `README.md` — додати рядок у таблицю команд/подій.
4. Game Hub shell — віддзеркалити протокол вручну.

Пастка: кроки 1 і 2 підтримуються окремо. TypeScript-union — це compile-time; `Set` у `bridge.ts` — це runtime. Оновиш union, але забудеш `Set` — і нова команда **проходить перевірку типів, але мовчки відкидається** в рантаймі через `isShellCommandType`. Я колись наступив саме на цей баг — ідеальний кандидат, щоб закодувати в команду.

## Що я додав у `memory-cards`

Дві Claude Code slash-команди в [`.claude/commands/`](https://github.com/smnvsergii/phaser-memory-cards-MFE/tree/main/.claude/commands):

### `/add-mfe-message`

```markdown
---
description: Add a new MFE protocol message (command or event) end-to-end...
argument-hint: <command|event> <messageType> [payloadShape]
allowed-tools: Read, Edit, Bash(npm run typecheck), Bash(npm run lint)
---
```

Тіло — нумерований чекліст усіх чотирьох кроків вище, з явно винесеною пасткою рантайм-`Set` і фінальним «нагадай мені віддзеркалити це в shell». Виклик:

```
/add-mfe-message event hintUsed { remaining: number }
```

### `/verify`

Тривіальна, але дуже часта: запускає `typecheck → lint → build` по черзі й рапортує
pass/fail. Економить набір послідовності й гарантує, що я ніколи не пропущу крок
перед тим, як назвати щось «готовим».

## Анатомія хорошої команди

- **Frontmatter робить обв'язку.** `description` (для пошуку), `argument-hint`
  (використання), `allowed-tools` (обмежує, що вона може запускати — `/verify`
  може запустити лише три npm-скрипти й нічого більше).
- **Аргументи роблять її перевикористовуваною.** `$ARGUMENTS` / `$1` / `$2`
  перетворюють разовий промпт на параметризований інструмент.
- **Кодуй пастку, а не лише щасливий шлях.** Уся цінність `/add-mfe-message` —
  у тому, що вона *пам'ятає про рантайм-`Set`*, те, що люди забувають.
- **Делегуй правилам.** Команди кажуть «слідуй `AGENTS.md`» замість повторення
  конвенцій, тож джерело істини й далі одне (див. 1.2).
- **Будь чесним щодо побічних ефектів.** `/add-mfe-message` відмовляється
  редагувати shell-репо й лише *позначає* крок дзеркалення — не вдає, що робить
  роботу, яку не має робити.

## Зауваження щодо портативності

Slash-команди за формою специфічні для інструмента: Claude Code використовує
`.claude/commands/`, Cursor — `.cursor/commands/` тощо. На відміну від правил
(де `AGENTS.md` — спільний стандарт, див. 1.2), єдиного крос-інструментального
формату для команд поки немає. Я тримаю *логіку* у файлі команди, а *конвенції*
в `AGENTS.md`, тож портування команди на інший інструмент — це переважно
копіювання + правка frontmatter.

## Результат

Процедура додавання повідомлення протоколу перетворилася з «відкрий чотири файли
й сподівайся, що згадаю про рантайм-`Set`» на один виклик, що робить це стабільно.
Команди перетворили мою найповторюванішу й найбагливішу рутину на одну безпечну дію.
